QSign API отдаёт не “магический перевод”, а проверяемый контракт

Разработчик получает sign-plan, coverage trace, сохранённый job, render-plan и AI-video brief. Эти данные удобно встраивать в редакционные, образовательные или accessibility-процессы.

Быстрый запрос

Минимальный text-to-sign вызов:

curl -X POST https://qsign.qdev.run/v1/translate/text \
  -H 'content-type: application/json' \
  -d '{"text":"Мне нужна помощь","language":"ru"}'

Ответ содержит `language`, `units`, `coverage`, `metadata.job_id` и признак сохранения в базе.

Правила интеграции

  • Не показывайте draft как сертифицированный перевод.
  • Отмечайте fallback и слова без словарного жеста.
  • Для медицины, права, финансов, ЧС и госуслуг требуйте человеческое ревью.
  • Не храните review token в клиентском JavaScript.
